## Notable Observations

### Phase architecture correctly processed by both responding models
Both Opus and Codex correctly identified the new Step 2c (Phase Transition Gate) and Step 2d (Phase 2 Entry) in their first-turn sequences. Neither model confused the new steps with the old architecture. This is the primary validation signal — the phase split is processable.

### Codex MFA clarity rating dropped (8 → 6)
Codex rated the candidate's MFA clarity at 6/10 (down from 8/10 on Arm B baseline). Cited reasons: forward references to undefined concepts, mixed terminology (research/investigation), and nested branching complexity. This suggests the Phase 1/Phase 2 addition increased perceived complexity for lower-capacity models even though the architecture is cleaner.

### Codex found a real spec contradiction
Codex flagged that the Step routing table marks Recall (1b) differently than the step prose. The table shows "—" for Small (correct) but the step header now says "all tasks" after our review fix. Worth verifying this is actually consistent.

### Candidate-specific failure mode: Phase Transition Gate
Both Opus and Codex identified Step 2c as a new failure point:
- **Opus**: The 4-way classification (research/code-change/ambiguous/plan-review) requires subjective evaluation that LLMs may get wrong
- **Codex**: The gate is behavioral (no SQL verification) — easy to skip entirely

### Limited statistical power
With only 2 models producing results (due to Agent Gate blocking), the comparison has very limited statistical power. The Opus flat score (49=49) is the strongest signal. The Codex -1 is within noise.
