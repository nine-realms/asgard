## Interpretation

**Overall: +0.2 avg (44.3 → 44.5).** The v0.9.4 structural reorganization delivered a small but real improvement, with the sole score change coming from GPT-5.4 (+1 on Failure Mode Quality). More importantly:

- **The MFA→Step 0 gap fix worked.** No model flagged the 110-line navigation hazard — the top finding from v0.9.3 is resolved. 3/4 models noticed the structural change but framed it as a trade-off (cleaner flow vs longer cross-references), not as a problem.

- **A new trade-off emerged.** Moving reference sections to end-of-file closed the navigation gap but introduced cross-reference distance. MFA step 1 says "output the Runtime Gate error message from the Runtime Gate section" — that section is now 580+ lines away. MFA step 2 says "Run the `CREATE TABLE` statement from the Verification Ledger section" — also at end of file. Fix #2 above addresses this by inlining the referenced content.

- **Convergence shifted to deeper issues.** The top findings are now about continuation detection fragility (4/4), start signal timing (4/4), and materialization complexity (4/4). These are the same issues that surfaced in v0.9.2-v0.9.3 — they represent the remaining ceiling for instruction-level improvements.

- **The file has stabilized at 44-45 avg (Good/Excellent boundary).** Four consecutive runs show diminishing returns from instruction-level changes. The remaining improvement opportunities are increasingly about runtime behavior (start signal timing, continuation detection) rather than instruction clarity.

**Key insight for next iteration:** Fix #1 (start signal timing) is the highest-impact remaining change — all 4 models agree the start signal should come earlier. Fix #2 (inline cross-references) would clean up the v0.9.4 trade-off. Together they'd likely push GPT-5.4 and Codex up by 1 point each.
