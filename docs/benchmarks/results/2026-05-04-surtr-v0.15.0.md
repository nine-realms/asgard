## Analysis

### Overall

v0.15.0 average (45.00) is the highest Surtr score since v0.14.0 (45.00). The v0.14.4 hardening fixes had caused a -1.50 regression (mostly Sonnet scoring variance); v0.15.0 fully recovers that and exceeds it. Three of four models improved. The codex regression (-5) is the main concern and is analyzed below.

### Sonnet +8 — Scoring Variance (Positive Direction)

The v0.14.4 Sonnet score of 40 was flagged in that run's analysis as scoring variance — the v0.14.2 Sonnet score of 49 used lenient standards, and v0.14.4 applied stricter standards to the same structural weaknesses. v0.15.0 Sonnet scored 48, applying strict standards (Q4 rated 5/10 by the model, justification scored 9 by rubric). The 40→48 move reflects that the v0.15.0 file has meaningfully addressed structural ambiguities that depressed the strict scoring: the 0c Cancel path with explicit SQL, the Frigg IN() gate form, the dead `1g` header removal. Sonnet's Q4/Q5 structural critiques are now sharper and more targeted, suggesting the file is easier to reason about even if the entry-point clarity rating remains strict.

### Codex -5 — Regression, Not Catastrophic

Codex scored 40 (v0.15.0) vs 45 (v0.14.4). The v0.14.4 codex improvement was tied to two specific changes: the inline change-approval definition and the ENTRY POINT sentinel. Those structural helpers are unchanged in v0.15.0. The regression is likely attributable to the 0c cancel path SQL (which adds two SQL statements and an `ask_user` to a path codex didn't previously need to trace) and the file-count guard (a new conditional that adds cognitive load). Codex is the model most sensitive to instruction length and conditional density — and v0.15.0 adds ~20 lines of conditional logic.

At 40/50, Surtr codex remains "Good" on the rubric (38-44 = "minor clarity improvements would help"). This is not a critical regression.

### Gate Identification +0.25 — Frigg Audit Trail Helps

The `IN()` gate form and distinct `review-frigg-approved` / `review-frigg-timeout` check_names made the Frigg gate semantics unambiguous. All four models correctly identified the IN() query and the exclusion of `review-frigg-timeout` (passed=0) from the gate. GPT-5.4 scored 10/10 on gates for the first time in a Surtr benchmark — both Gate 3a's SQL and the non-SQL "user must respond" clause were correctly identified.

### Entry-Point Clarity +0.50 — 0c Cancel Path Clarification

The explicit 0c cancel SQL (INSERT `task-complete` with tool=`user-cancel`) gave models a clearer mental model of the cancellation path's effect on the ledger. Both Opus and Sonnet cited this as making the "orphan task" problem visibly solved. The dead `1g` header removal reduced noise in the step flow. Combined, these improved Entry-Point Clarity by half a point despite Sonnet's still-strict 5/10 rating.

### Cross-Model Convergence

All four models identified the same two top structural risks:

1. **Router / FORCE boundary** — semantic routing for borderline messages, "first turn" ambiguity in the FORCE rule (does "first turn" mean first message of session or first tool-call turn per loop entry?)
2. **Step 0a continuation check** — the prose qualifier "Low-info entry only" is not structurally enforced; LLM following step 0 linearly may run 0a on any message, then exit to Conversation mode if no open task found

Surtr-specific finding this run (3/4 models): **Step 5c Small ambiguity** — the routing table shows Small gets `—` for 5c, but the Step 5c section itself has no explicit "if Small, skip this entire step" clause in the step body. The omission is by design (task size is known before Step 5c), but models flagged it as a compliance risk.

## Per-Model Summary

### claude-opus-4.6 (49/50, unchanged from baseline)

Stable at ceiling. 13-step sequence with complete 1x coverage, recall availability guard SQL, Frigg with full timeout/approved paths, 3c baseline. Q2: table with 5 explicit gates plus structural write backstop. Q3 standout: FM2 (3a ask_user gate "continue generating" momentum — SQL count passes green, LLM proceeds without pausing for user input) is the most operationally specific finding. Q5: 7-step critical path with each step named; 3 of 7 are judgment calls without structural enforcement.

### claude-sonnet-4.6 (48/50, +8 vs baseline)

Highest Claude Sonnet Surtr score to date. Q1 covers the 0c Cancel path explicitly including the new INSERT SQL and both `ask_user` cancel options — one of two models to identify the cancel SQL by name. Q2: 7 gate sections (Runtime, Loop-entry, Frigg, Baseline, 5c, 5e, Step 8) with correct thresholds and the `review-frigg-timeout` exclusion called out. Q3: FM3 (Small 5c ambiguity) is the most actionable — correctly identifies that the step body has no "if Small → skip" directive. Q4: 5/10 entry-point rating (strict, same pattern as Odin run) with 5 well-documented structural problems. Q5: 10-step critical path analysis with ✅/⚠️ annotations; identified the Frigg resume turn (user says "proceed" after gate) as the highest-risk continuation path.

### gpt-5.4 (43/50, +3 vs baseline)

Solid improvement. Q1 matches Opus in completeness. Q2: 11 items (most in this panel) — includes runtime gate, loop-entry verify, write backstop, evidence-bundle prerequisite, and recall availability guard as non-emoji hard stops. The observation that "loop-entry is a write-backstop, not a routing enforcer — it doesn't force entry into Step 0 if routing already went to Conversation" is the sharpest structural insight from GPT-5.4 in this run.

### gpt-5.3-codex (40/50, -5 vs baseline)

Regression attributed to added conditional density (0c cancel SQL, file-count guard). Q1 covers the core path but is thinner on 1x substeps and recall. Q2: 5 gates listed (correct but incomplete — missing runtime gate, loop-entry verify, write backstop). Q3 failure modes solid but not deeply non-obvious. This run's codex score (40) is still within the "Good" range and above the v0.14.1 caveman-prose low (38). The regression does not meet the threshold for blocking the release.

## Notes for Future Runs

- **"First turn" FORCE scope:** Clarify to "first tool-call turn per Surtr Loop / Ship entry, including resume turns." One phrase change, closes the 3/4 model finding.
- **Step 0a structural separator:** Replace prose qualifier with a sub-heading `### 0a — Continuation check (low-info only)`. Same change recommended for Odin.
- **Step 5c Small skip:** Add a one-line "Small: skip Step 5c entirely" at the top of the Step 5c section body, before the Medium/Large sub-sections. Removes ambiguity without changing behavior.
- **Codex compliance recovery:** The codex regression is most likely from added conditional density. If codex scores remain below 42 in the next benchmark, consider whether the 0c cancel SQL can be made more telegraphic.
- **Sonnet Q4/Q5 structural issues:** The "EXECUTE block reads as summary not directives" and "GUARD is passive framing" findings from v0.14.4 remain open — neither was resolved by v0.15.0 changes. Target for a future compression pass.
