## Changes under test (v0.10.15)

- **Continuation `task_id` hardening**: MFA Step C now explicitly discards the prior `task_id` on both new-task exits, Step D states that every new-task path generates a fresh slug, and the Verification Ledger clarifies that only true continuations reuse MFA Step C's value.
- **Evidence Bundle readiness tightening**: Step 5e now uses `COUNT(DISTINCT check_name)` and excludes `tier3-infeasible`, so duplicate rows and infeasibility bookkeeping no longer satisfy readiness.

---

## Notable Findings

### Opus — 50/50 (perfect)
- The exact v0.10.14 complaint about 5e counting raw rows is gone. Opus now reads the readiness gate as materially tightened.
- New highest-risk finding is **silent startup-step collapsibility**: Steps 0b–2 still rely on a single `context-gathered` sentinel rather than per-step evidence, so an LLM can skip them and self-report completion.
- Continuation routing is still the most delicate stateful branch in the file, but the explicit discard/fresh-slug language removed the prior shadowing complaint.

### Sonnet — 48/50
- The stale-`task_id` failure mode remains the top risk area, but the benchmark no longer reports the prior **step-C task_id shadowing ambiguity** as an unresolved spec bug. The file now clearly says new-task paths discard the carried-forward ID.
- New main complaint: **start signal timing**. The `🔁 Starting...` line sits behind ambiguity/pushback resolution, so a pre-signal `ask_user` can make the loop feel like it never started.
- Step 5c remains the mechanically hardest operation in the spec; Sonnet still treats prompt materialization/order-of-operations as the main downstream compliance risk.

### GPT-5.4 — 43/50
- Picked up the tightened 5e query correctly, including the `tier3-infeasible` exclusion.
- Still sees two "starts": internal loop-entry success vs. later user-visible `Starting...`, which keeps Q5 below Claude levels.
- Notes Step 3c / baseline gating and overall startup sequencing more accurately than v0.10.14, giving a small recovery on Q3.

### Codex — 40/50
- No regression, but no real gain either.
- Still dominated by MFA sequencing, continuation/state complexity, and Step 5c prompt-materialization load.
- Correctly treats 5e as a distinct-check gate now; the old raw-row concern did not reappear.
