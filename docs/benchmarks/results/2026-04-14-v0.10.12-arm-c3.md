## Key Findings

### Confirmed improvements from v0.10.12
- **Opus +4** and **Sonnet +2**: The MFA→Loop boundary fix clearly helped both Anthropic models. No model mentioned the old "MFA complete. New tasks → begin the Odin Loop at Step 0." navigation sentence as a stop point. The inline verify in MFA-E was correctly identified and described by all models.
- The SELECT 1 duplication complaint (from C vs C2) remains absent.
- All 4 models correctly located the loop-entry verify as part of MFA-E (not a standalone Step 0 gate), confirming the Gate Registry update is consistent with the spec.

### New findings for next iteration

**Finding 1 — Start-signal vs ambiguity/pushback gate ordering (Opus, high confidence)**
Opus identified that the ordering between sizing, start signal, and ambiguity/pushback gates in Step 0 is underspecified. The spec says the start signal fires "immediately after sizing" but the ambiguity gate fires "after boosting" — if boost → ambiguity gate → size → start signal, the user sees a question with no start signal. Concrete fix: add numbered sub-steps to Step 0 specifying the exact order.

**Finding 2 — "The SELECT 1 from the first batch above" is a forward reference (Sonnet, high confidence)**
Step A says "The `SELECT 1` from the first batch above" but at the time the LLM parses Step A, it hasn't executed the first batch yet. The bold paragraph defining the first batch appears before the lettered steps, so the reference is technically valid — but it's a structural disconnect that creates parsing ambiguity. Concrete fix: change to "Run `SELECT 1` (session DB)" with no back-reference.

**Finding 3 — Size escalation during planning has no re-run trigger (Sonnet, medium confidence)**
If a file discovered during plan drafting (Step 3) has a 🔴 risk tag, the spec says to reclassify and re-run Steps 1b+2 at the new depth. But this instruction is buried in a conditional paragraph mid-Step 3 after Steps 1b and 2 have completed. Models that sized Medium and ran 2-3 searches are unlikely to backtrack. Fix: add an explicit re-run gate: "If reclassified → rerun 1b+2 before proceeding to 3a."

**Finding 4 — No programmatic barrier between MFA and Step 0 (Sonnet + GPT-5.4)**
MFA is entirely silent. An LLM that partially completes MFA and drifts toward responding can bypass MFA with no observable consequence until Step 0's precondition fires. The Step 0 catch ("if you somehow reached Step 0 without a loop-entry row...") is a weak recovery — it relies on the model noticing an absent row. Concrete fix (Sonnet suggested): add `---` hard separator + `▶ BEGIN LOOP` marker at MFA/Step-0 boundary.

### Codex stochastic dip
Codex dropped 5 points from C2 (42→37). The response was notably more compressed than prior Codex runs — correct but less structurally detailed. This is likely stochastic variance rather than a regression caused by v0.10.12 changes; no model mentioned new structural problems introduced by this version.

## Verdict

v0.10.12 is a **marginal improvement** overall (+0.25 avg), but the per-model story is encouraging:
- Anthropic models improved meaningfully (+2 to +4)
- GPT-5.4 held flat (no regression)
- Codex dropped (likely stochastic)

The benchmark surfaced 4 new actionable findings (Findings 1–4 above) that would be strong candidates for v0.10.13. The instruction structure is now stable enough that all models correctly describe the MFA→Loop flow — the remaining score gap is from structural ambiguities deeper in the flow (Step 0 sub-step ordering, Step 3 size-escalation backtrack).

Safe to commit v0.10.12.
