## Regression Note — v0.14.3

The round-2 DELETE introduced in v0.14.2 used `check_name LIKE 'review-%'`, which matches `review-frigg` (also `phase='review'`). After a round-2 re-run, the Step 8 pre-commit re-check of the 5c gate would pass (reviewer rows freshly inserted), but the Frigg row evidence would be silently destroyed. Fixed in v0.14.3 by replacing the LIKE pattern with an explicit IN list of the 10 reviewer check names.

## Notes for Future Runs

- **Size escalation gate** (sonnet Q3): add a check at Step 4 entry — "if any file touched is 🔴, re-classify to Large, return to Step 3a." One sentence; closes a real compliance gap.
- **Conversation = FORCE bypass** (gpt-5.4 Q3): FORCE currently only listed under Surtr Loop/Ship. Adding it to Conversation header ("No `report_intent` + `SELECT 1` in Conversation — read-only only") makes the asymmetry explicit rather than implicit.
- **Section-jump** (opus Q5): A `> ⚠️ **ENTRY POINT** — process On Every Message before any section below` sentinel above the routing block would close this path.
- **"Change approval" undefined** (opus, codex): Step 0a's `No live: change approval → Fresh. Else → Talk.` — "change approval" is never defined. One example would close this: `(change approval = low-info reply directly following a code-change plan or task description)`.
- **Observable route signal**: emitting `> 🔥 Route: {Surtr Loop}` after classification makes routing debuggable without adding compliance cost.
