### Failure Mode Quality

### MFA Clarity

- **Others (8):** All gave 7-8 ratings with reasonable justification. Common theme: "do not read further" is unenforceable for LLMs.

### Loop Start Diagnosis
- **Opus/Sonnet (10):** Both identified 4+ specific structural causes. Opus proposed a commitment device (`⏳ Initializing Odin...`). Sonnet identified task_id reuse as a silent bypass and Investigation misclassification as the most likely observable symptom.
- **GPT-5.4 (9):** Solid 3-cause analysis with good "not crisp enough" summary
- **Codex (8):** Correct but less detailed

## Convergent Findings (2+ models)

1. **Prompt materialization order inconsistency** (Codex + GPT-5.4): Agent says resolve models → conditionals → remaining; skill says conditionals → variables. Practically harmless (model vars don't appear inside conditionals) but a spec mismatch that could confuse an LLM.

2. **"Do not read further" is unenforceable** (all 4): LLMs process full context before generating. The instruction models sequential reading but execution is parallel.

3. **New-task vs continuation misclassification** (all 4): The 3-check rule is a reasoning judgment with no hard enforcement. All models identified this as the primary loop-start vulnerability.

4. **Start signal delayed** (Opus + Sonnet + GPT-5.4): Visible loop start comes after many silent steps — perceived "non-start" even when MFA executed correctly.

## Novel Finding (single model)

- **Sonnet:** Task_id reuse on misclassified continuations silently passes the loop-entry gate. Old task's `loop-entry` row exists, so the gate passes even though MFA never ran for the new task. This is a structural gap no other model identified.

## Analysis

Average 44.0 — solidly in the "Good" band (38-44) and within normal variance of the v0.9.4 high-water mark (44.5). The Mimir-on-Small changes added 2 new gate variants that all models except Codex correctly identified and described.

**Codex continues to be the weakest:** 39/50 in this run, consistent with prior benchmarks (37-40 range). Missing the loop-entry gate is a significant blind spot. This is relevant because Codex (`gpt-5.3-codex`) is used as Tyr's primary model and Heimdall's model — both review tasks where gate awareness matters less (reviewers don't execute gates).

**GPT-5.4 improved from 44 to 45**, correctly enumerating all 11 scored gate variants — the best gate identification in this run.

**Sonnet continues to produce the deepest failure analysis**, with the novel task_id reuse finding being the standout insight of this benchmark run.

No regressions detected from the Mimir-on-Small changes.
