# Benchmark: v0.13.0 Structural Startup Consolidation

**Date:** 2026-04-17
**Branch:** `odin/startup-path-simplification`
**Baseline:** v0.12.3 (main, 593 lines)
**Candidate:** v0.13.0 (618 lines)
**Protocol:** Standard 5-question simulation (explore agents, full payload with 3 skill files)
**Models:** GPT-5.4, Opus 4.5, GPT-5.3-Codex, Sonnet 4.5

## Change Summary

Replaced 6 overlapping routing authorities (Intent Router table, canonical dispatcher, forcing rule, hard invariant, fail-closed default, code-change handoff) with one 4-step dispatch algorithm. Moved continuation logic from the router into Step 0a/0b. Simplified Conversation mode to a one-liner safety guard. Added contrastive examples table.

## Analysis

**Result: Marginal positive.** Average moved +0.75 (44.75 → 45.50). Opus showed the clearest signal (+2, driven by better Q1 step completeness and Q2 gate enumeration). Codex gained +1 on Q4 (rated candidate clarity 8/10 vs baseline 9/10 — the baseline was overgenerous). GPT-5.4 and Sonnet 4.5 were flat.

**Why the gain is small:**

1. **Sonnet 4.5 ceiling effect.** Scored 49/50 on both arms. The file is already clear enough for Sonnet — it has no room to improve. This model is not the bottleneck.

2. **GPT-5.4 structural awareness.** Both arms produced nearly identical 18-22 step sequences. GPT-5.4 parsed the old distributed authority fine — it just happened to navigate it correctly. The consolidation didn't unlock new compliance.

3. **The real bottleneck is downstream.** All 4 models identified the same top-3 failure modes regardless of arm: (a) loop-entry verification skip, (b) 3a/3c gate sequencing, (c) 5c materialization complexity. The startup rewrite doesn't touch any of these.

**What moved:**

- **Opus Q1/Q2** improved because the candidate's single routing table + 4-step algorithm was easier to enumerate completely. Baseline Opus listed 16 steps; candidate listed 17 with explicit 0a skip reasoning.
- **Codex Q4** improved because the candidate's explicit "First action" + 4-step algorithm reads more clearly than the baseline's scattered routing rules. Codex baseline overrated clarity at 9/10; candidate correctly rated 8/10.

**Carry-forward insights:**

- The structural consolidation is a clean improvement but not a benchmark mover. Ship it for code quality, not for score gains.
- Future optimization should target Step 3c baseline gate (self-referential wording), Step 5c materialization (6-phase complexity), and the "soft HALT" pattern (gates that say "retry" but don't enforce STOP).
- Sonnet 4.5's 49/50 establishes a practical ceiling for this benchmark protocol. To differentiate further, the protocol itself may need harder questions.
