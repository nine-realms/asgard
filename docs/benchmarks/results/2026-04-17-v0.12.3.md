## Changes in v0.12.3

Prose compression of two redundant reference tables (−27 lines, 668 → 641):

1. **Subagent Strategy table → prose** — 7-row Agent/Use When/Anti-pattern table compressed to 2 paragraphs. All behavioral signals preserved; cross-references to Steps 3a and 5c for full agent details.
2. **Step Routing table → exceptions summary** — 14-row size matrix compressed to a single sentence listing only the exceptions (Small skips 3c/5d/5e/ledger, Medium skips 5d, reviewer composition by size, Step 6 Small limitation).

Gate Registry table intentionally kept — serves as audit checksum validated by `make check`.

## Score Movement Analysis

- **Codex +3**: Biggest gainer. The shorter file improved parse quality across all dimensions. MFA Clarity jumped from 6 → 8 — the reduced table density made the file feel more structured to this model.
- **GPT-5.4 ±0**: Stable. Compression had no measurable impact on this model's analysis depth.
- **Sonnet −3**: Dropped entirely on Q4 MFA Clarity (9 → 6). The compressed tables removed visual landmarks that Sonnet used as clarity signals. However, Q1 (10) and Q2 (10) were unchanged — comprehension was not affected, only the subjective clarity rating.
- **Opus −3**: Similar pattern to Sonnet. Q4 dropped from 10 → 7. Same root cause: removal of tabular structure reduced perceived clarity without affecting actual gate/sequence comprehension (Q1: 10, Q2: 10 unchanged).

**Key insight:** The Q4 drops cite pre-existing issues (Runtime Gate physical distance, no "START HERE" anchor, 118 lines of preamble) — not the compression itself. The tables may have served as visual "structure cues" that masked those deeper issues.

## New Findings (v0.12.3 run only)

- **Opus:** Identified the continuation handler as the most likely cause of "loop not starting" — `ask_user` → user replies "just do it" → continuation handler finds no open task → treats as conversational acknowledgment. Sharp diagnosis.
- **Opus:** Suggested structural guard: "If message contains fix/add/refactor/update verbs → default to Odin Loop unless clearly a question ABOUT the action."
- **Sonnet:** Noted Step 8 meta-gate re-verifies 3a+3c+5c gates — good defense-in-depth but means earlier gate skips aren't caught until commit time.
- **Sonnet:** Frigg timeout row (`review-frigg-timeout`) is "bookkeeping, not approval" but an LLM under token pressure could confuse it with a passed gate.
- **Codex:** MFA analysis improved significantly — correctly identified multi-mode routing as the main risk to first-action clarity.
