## Analysis

**Average 45.0 — first time crossing into the "Excellent" band (45-50).** The +1.0 gain is entirely driven by Codex (+3) and Sonnet (+1), with no regressions on any model.

**Codex breakthrough (39→42):** The largest single-version Codex improvement. Two fixes contributed:
- **Fix A** (gate formatting): Standardizing the loop-entry gate to `🚫 GATE:` pattern let Codex find all 11 gates for the first time (Gates 7→9, +2)
- **Fix C** (Step 0 labels): Adding bold labels to previously unlabeled sub-actions improved step enumeration (Sequence 8→9, +1)
- **Fix B** (Gate Registry) likely contributed to the Gates improvement by providing a single scannable table, though it's hard to isolate from Fix A

**Sonnet new all-time high (48):** The consistent gate formatting appears to have helped Sonnet's already-strong gate identification push from 9→10.

**Continuation recovery (Fix E) is not benchmark-testable** — it addresses a runtime failure mode (context compaction mid-task) that single-turn benchmark simulations don't exercise. Its value is validated by the real-world Sonnet failure that motivated it.

**Step 8 gate fix (Fix F, Mimir catch)** standardized the last remaining non-standard gate heading. Not directly scored since Step 8 was already identified by all models, but removes a potential future regression if models begin pattern-matching more strictly.

**Codex ceiling estimate:** With formatting inconsistencies resolved, Codex's remaining gap (42 vs 45-48) is likely inherent to the model's depth of analysis rather than spec scannability. Further Codex gains would require either model improvements or fundamentally different spec structures (e.g., structured data formats instead of prose).
