## Analysis

### Fix Validation

All three fixes landed correctly in the instruction file:

- **Fix 1 ✅**: 4/4 models placed start signal after sizing, before ambiguity gate in their Q1 sequence. No model flagged "start signal is too late" (the top v0.9.4 complaint). The v0.9.4 finding is resolved.
- **Fix 3 ✅**: 1/4 models (GPT-5.4) explicitly noted the uncertainty default. The others processed it without comment — consistent with it being a natural part of the continuation rule.
- **Fix 5 ✅**: 1/4 models (Sonnet) explicitly referenced size escalation during Step 3 plan drafting. The instruction is being picked up.

### Score Movement

The −1.25 average drop is within model variability (±2 points per model per run). No structural regression was detected:

- **Opus +1**: Only model to improve. Found all 9 gates (perfect 10), identified the Step 0 gate recovery path issue (new finding). Opus is the most reliable benchmark signal.
- **Sonnet −2**: Internal contradiction in Q5 — correctly sequenced sizing→signal→ambiguity in Q1 but claimed ambiguity gate could fire before signal in Q5. Deducted 1 point on Q5 for this.
- **Codex −2**: Less detailed failure mode and MFA clarity analysis than v0.9.4. Model variability — no new issues identified.
- **GPT-5.4 −2**: Slightly less detailed Q4/Q5 analysis. Correctly noted start signal is "after sizing inside Step 0" as improved but still a concern if boosting stalls.

### New Findings Worth Investigating

1. **Step 0 gate recovery path (Opus, 1/4)**: The loop-entry gate at Step 0 says "go back to MFA step 4 and INSERT the loop-entry row." But if you reached Step 0 without running MFA at all, step 4 is too narrow — you need the full MFA restart from step 1 (Runtime Gate, CREATE TABLE, etc.). Fix: change "go back to MFA step 4" to "go back to MANDATORY FIRST ACTIONS step 1 and execute all steps."

2. **No MFA completion signal (Opus, 1/4)**: MFA has no observable "done" marker. Adding `> ✅ MFA complete` after step 4 would create an auditable checkpoint. Counter-argument: the loop-entry INSERT is the durable marker, and a visible signal would violate the "minimal output" directive for Steps 0-2.

3. **Start signal still blocked by pre-sizing stall (Sonnet/GPT-5.4, 2/4)**: Even with Fix 1, if the model stalls during boosting (before sizing), the start signal never fires. This is a narrower version of the v0.9.4 concern — the window of silence is now just the boost step, not boost+ambiguity+pushback.

### Recommendations

| # | Fix | Convergence | Risk | Recommendation |
|---|-----|-------------|------|----------------|
| 1 | Widen Step 0 gate recovery to full MFA restart | 1/4 | Low | **Do it** — surgical text change, closes a real hole |
| 2 | Add MFA completion signal | 1/4 | Low | **Skip** — conflicts with minimal-output directive, loop-entry INSERT is sufficient |
| 3 | Move start signal before sizing (into MFA) | 2/4 | Medium | **Skip for now** — would require removing `{size}` from signal or moving sizing into MFA. Current position (after sizing) is a good balance. |
