## Analysis

### Overall

The 4 v0.14.4 fixes produced mixed results: codex improved sharply (+5), opus held (49), and GPT-5.4 dipped slightly (-2, within variance). The headline regression is sonnet: 49 → 40 (-9). The delta is explained by scoring variance, not code regression — sonnet's v0.14.2 Q4/Q5 scores (9, 10) were generous on structural weaknesses that have been present since before v0.14.2. The clean-run evaluation in v0.14.4 was materially more critical on the same issues (Q4=7, Q5=7).

The 4 fixes did land for codex. Q3 improved 7→9 (the change-approval definition clarified a failure mode), and Q4 improved 7→8 (the ENTRY POINT sentinel made the entry path clearer). These are the models that most benefit from explicit markers — Claude models already handle implicit routing correctly and aren't rewarded by the rubric for doing so.

### Sonnet Q4/Q5 drop — scoring variance, not regression

Sonnet's v0.14.2 Q4=9, Q5=10 scores reflected a lenient reading. The same structural weaknesses it identified in v0.14.4 (EXECUTE block is summary not directives; passive GUARD framing; Step 0a "Low-info only" label makes fresh-path routing implicit) were present in v0.14.2. The v0.14.4 evaluation applied stricter standards: Q4=7, Q5=7. These issues are known and unfixed — they are candidates for a future compression pass.

### Codex +5 — fixes directly targeted its gaps

The codex improvement is real:
- Q3: 7 → 9. The change-approval inline definition (Step 0a) closed the "change approval undefined" finding that codex had flagged in v0.14.2.
- Q4: 7 → 8. The ENTRY POINT sentinel gave codex a clear entry marker to reference.
- Q5: 7 → 9. Improved loop-start diagnosis depth — both fixes contributed to a cleaner flow narrative.

### Gate Identification drop — structural, not catastrophic

Q2 dropped from 10.00 to 9.50. Sonnet (9/10) specifically called out Gate 8 (pre-commit re-check) as a compliance risk: the instruction says "re-run Step 5c gate" but doesn't repeat the query inline, requiring the model to look back. This is a real issue — the highest-stakes gate is the least explicit.

## Per-Model Summary

### claude-opus-4.6 (49/50, unchanged)

Stable at the ceiling. Complete 5-gate enumeration. Sequence includes correct 0a stale-burn + live-task query path for the login-bug scenario (routes directly to 0b, skips 0a). No new structural findings beyond known issues.

### gpt-5.3-codex (45/50, +5 vs v0.14.2)

Biggest mover this run. Q1 precise: correctly named `report_intent("Lævateinn rises")` + `SELECT 1` as mandatory first calls before any prose. Q2 perfect: 5 gates with exact queries and thresholds, plus called out loop-entry and runtime probe. Q3 improved: added bookkeeping drift (check-name/phase mismatch causing gate deadlock or false pass) as third failure mode — not flagged in prior runs. Q4=8: ENTRY POINT sentinel helped; routing ambiguity persists. Q5=9: fragile at routing + FORCE boundary, correctly identified the specific routing failure paths.

### claude-sonnet-4.6 (40/50, -9 vs v0.14.2)

Drop is scoring variance — this run was materially stricter on Q4/Q5. Key Q5 finding (new): Step 0a is labelled "Low-info entry only" which creates an implicit question about what handles direct code-change requests. The answer (they skip 0a and go to 0b Fresh) is correct but unstated — a model unfamiliar with the pattern must infer it. Q2 finding: Gate 8 should repeat the 5c query inline rather than back-referencing. Q3 finding: Frigg rerun-after-plan-change trigger has no detection mechanism — if implementation diverges from the approved plan, nothing forces a re-run.

### gpt-5.4 (40/50, -2 vs v0.14.2)

Within scoring variance. Persistent Q4/Q5 gap: router decides before FORCE fires, so FORCE is reactive not proactive. Q2=9: same Gate 8 concern as sonnet.

## Notes for Future Runs

- **Step 0a label** (sonnet Q5): "Low-info entry only" should add a parenthetical: `(direct code-change requests skip to 0b Fresh)`. One line, closes the implicit routing gap.
- **Gate 8 inline query** (sonnet, gpt-5.4): repeat the 5c gate query inline at Step 8 instead of cross-referencing. Highest-stakes gate should be most explicit.
- **EXECUTE as directives** (sonnet, codex): consider expanding the EXECUTE bullet for Surtr Loop from `→ Step 0: …` to something that signals "this is a command, not a description."
- **Sonnet Q4/Q5 variance**: known structural issues in entry-point clarity and FORCE brittleness remain open. Target for a future pass if overall average needs to recover.
