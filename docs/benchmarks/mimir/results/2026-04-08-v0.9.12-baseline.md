## Key Findings

### What v0.10.0 improved
1. **Strongest models benefited most.** Opus gained +2 and GPT-5.4 gained +1 — the additional architectural material (companion skill, confidence filter) gave them more to analyze and reason about in Q2 (Lanes) and Q5 (Failure).
2. **No model was hurt.** Sonnet and Codex scored identically — the structural changes didn't confuse or degrade any model's comprehension.
3. **Inline CCA doesn't hurt comprehension.** All models correctly processed the 23 inline heuristics. The extraction to a companion skill provides organizational benefits (shorter core prompt, conditional loading) but doesn't measurably improve instruction following.

### What v0.10.0 didn't improve
1. **Codex ceiling is analytical depth, not spec structure.** At 35/50 on both versions, Codex's limitation is synthesis capability, not instruction parsing. No spec restructuring will fix this.
2. **Sonnet is already at its ceiling.** 45/50 on both versions suggests Sonnet extracts maximum value from the spec regardless of structure. Gains would require richer spec content, not better organization.

### Novel per-model observations (v0.9.12 only)
- **Opus:** Model selection × file length interaction for reliability; CCA-009 (idempotency safety) as a firing heuristic others missed
- **Sonnet:** 23 heuristics exceeding spec's own ≤20 threshold; "senior engineer" test calibrated per-model
- **GPT-5.4:** "Lane deference, not a confidence taxonomy" — precise characterization of v0.9.12's suppression model; diff rule vs omission-analysis tension as a structural failure mode
